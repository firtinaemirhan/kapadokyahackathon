# Tortu Autopilot Spec

Build a demo-ready MVP rather than waiting on external account setup. The app will be Supabase-ready and production-shaped, but local demo pages will tolerate missing env values through typed fallback data and external API fallbacks. Core proof points: dynamic server-side distance route, TCMB EVDS/today.xml exchange route with stale handling, independent geocoding/Overpass routes, carbon calculation endpoint, bilingual UI, listing search/detail, create wizard, dashboard/wallet/admin, certificate PDF response, schema/seed, README.
